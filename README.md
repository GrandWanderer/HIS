# HIS (Node.js) — під ЛР №7 (UNIT tests)

Мінімальний “живий” шматок Hospital Information System (HIS) на **JavaScript (Node.js)** з чітким розділенням:
- **utils** (сервісні/утилітарні функції)
- **services** (бізнес-логіка)
- **data** (шар збереження даних — in-memory репозиторії)

Проєкт зроблено спеціально для **ЛР №7**: написати unit-тести для 3–5 методів із нетривіальною логікою та показати coverage.

## Зв’язок з вимогами (ЛР №1)
- **R1.3** Календар запису (конфлікти лікаря/кабінету, тривалість, минулий час)
- **R1.4** SMS-нагадування за 24 години (логіка часу надсилання)
- **R1.5–R1.8** Оплати (cash/card/POS, валідації)
- **R1.9** Ролі/права доступу (RBAC)
- **R1.13** Імпорт пацієнтів з CSV

## Структура
- `src/utils/*` — сервісні функції (валидації, час)
- `src/services/*` — бізнес-логіка (саме тут 3–5 методів для unit-тестів)
- `src/data/*` — “збереження даних” (репозиторії; легко замінити на БД пізніше)
- `test/*` — unit-тести

## Запуск
```bash
npm test
npm run coverage
```
